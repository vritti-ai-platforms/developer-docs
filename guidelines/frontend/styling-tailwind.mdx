Vritti uses Tailwind CSS v4 with a comprehensive design system. This guide covers styling patterns and best practices.

## Design System Tokens

### Color Tokens

All colors use semantic design tokens. Never hardcode hex, rgb, or hsl values.

```typescript
// Background colors
<div className="bg-background" />      // Base background
<div className="bg-card" />            // Card surfaces
<div className="bg-muted" />           // Subtle backgrounds
<div className="bg-primary" />         // Brand primary
<div className="bg-secondary" />       // Secondary actions
<div className="bg-accent" />          // Accent highlights

// State colors
<div className="bg-destructive" />     // Error/danger
<div className="bg-warning" />         // Warning state
<div className="bg-success" />         // Success state

// Text colors
<div className="text-foreground" />         // Primary text
<div className="text-muted-foreground" />   // Secondary text
<div className="text-primary" />            // Brand text
<div className="text-destructive" />        // Error text
```

### Color with Opacity

Use opacity modifiers for subtle backgrounds:

```typescript
// Success with 15% opacity
<div className="bg-success/15 text-success" />

// Destructive with 20% opacity
<div className="bg-destructive/20 text-destructive" />

// Primary with 10% opacity
<div className="bg-primary/10 text-primary" />
```

### Border Colors

```typescript
<div className="border border-border" />           // Default border
<div className="border border-input" />            // Input borders
<div className="border border-destructive" />      // Error borders
<div className="ring-2 ring-ring" />               // Focus rings
```

## State Styling

### Interactive States

```typescript
<button
  className={cn(
    // Base styles
    'rounded-md px-4 py-2 font-medium transition-colors',
    // Hover state
    'hover:bg-primary/90',
    // Focus state
    'focus:outline-none focus:ring-2 focus:ring-ring focus:ring-offset-2',
    // Active state
    'active:scale-95',
    // Disabled state
    'disabled:opacity-50 disabled:pointer-events-none',
  )}
>
  Click me
</button>
```

### Form Validation States

```typescript
<input
  className={cn(
    'w-full rounded-md border px-3 py-2',
    // Default state

    // Error state

    // Success state
    isValid && 'border-success focus:ring-success',
  )}
/>
```

## Class Name Utilities

### The `cn` Helper

Use the `cn` utility from quantum-ui for conditional classes:

```typescript
import { cn } from '@vritti/quantum-ui/utils';

const Button = ({ variant, disabled, className, children }) => {
  return (
    <button
      className={cn(
        // Base styles (always applied)
        'rounded-md px-4 py-2 font-medium',
        // Variant styles
        variant === 'primary' && 'bg-primary text-primary-foreground',
        variant === 'secondary' && 'bg-secondary text-secondary-foreground',
        variant === 'outline' && 'border border-input bg-background',
        // Conditional styles
        disabled && 'opacity-50 cursor-not-allowed',
        // Allow override via className prop
        className,
      )}
      disabled={disabled}
    >
      {children}
    </button>
  );
};
```

### Implementation

```typescript
// cn utility implementation
import { clsx, type ClassValue } from 'clsx';
import { twMerge } from 'tailwind-merge';

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

## Best Practices

<Accordion title="Never Hardcode Colors">
Always use design tokens, never raw color values:

```typescript
// ✅ Good
<div className="bg-primary text-primary-foreground" />
<div className="bg-success/15 text-success" />

// ❌ Bad
<div className="bg-[#3b82f6] text-white" />
<div style={{ backgroundColor: '#22c55e' }} />
```
</Accordion>

<Accordion title="Use Semantic Tokens">
Choose tokens based on purpose, not appearance:

```typescript
// ✅ Good - semantic meaning
<span className="text-destructive">Error message</span>
<span className="text-muted-foreground">Helper text</span>

// ❌ Avoid - appearance-based
<span className="text-red-500">Error message</span>
<span className="text-gray-500">Helper text</span>
```
</Accordion>

<Accordion title="Mobile-First Approach">
Start with mobile styles, add larger breakpoints:

```typescript
// ✅ Good - mobile first
<div className="p-4 md:p-6 lg:p-8" />

// ❌ Avoid - desktop first
<div className="p-8 sm:p-6 xs:p-4" />
```
</Accordion>

<Accordion title="Avoid Deep Nesting">
Keep class lists readable:

```typescript
// ✅ Good - use cn() for conditional classes
<div className={cn(
  'base-styles',
  condition && 'conditional-styles',
)} />

// ❌ Avoid - inline ternaries
<div className={`base-styles ${condition ? 'styles-a' : 'styles-b'} ${other ? 'more' : ''}`} />
```
</Accordion>
