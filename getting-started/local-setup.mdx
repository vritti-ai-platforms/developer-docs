This guide walks you through cloning the Vritti repositories and setting up your local development environment.

## Directory Structure

We recommend organizing all Vritti projects in a single directory:

```bash
mkdir -p ~/Documents/Vritti
cd ~/Documents/Vritti
```

## Clone Repositories

Clone the repositories you need based on your work:

<Tabs>
  <Tab title="Full Stack">
    ```bash
    # Shared packages
    git clone https://github.com/vritti-hub/quantum-ui.git
    git clone https://github.com/vritti-hub/api-sdk.git

    # Backend
    git clone https://github.com/vritti-hub/vritti-api-nexus.git

    # Frontend
    git clone https://github.com/vritti-hub/vritti-web-nexus.git
    git clone https://github.com/vritti-hub/vritti-auth.git
    git clone https://github.com/vritti-hub/vritti-landing.git
    ```
  </Tab>
  <Tab title="Frontend Only">
    ```bash
    git clone https://github.com/vritti-hub/vritti-web-nexus.git
    git clone https://github.com/vritti-hub/vritti-auth.git
    ```
  </Tab>
  <Tab title="Backend Only">
    ```bash
    git clone https://github.com/vritti-hub/vritti-api-nexus.git
    ```
  </Tab>
</Tabs>

## Install Dependencies

Install dependencies for each project:

```bash
# quantum-ui
cd ~/Documents/Vritti/quantum-ui
pnpm install

# api-sdk
cd ~/Documents/Vritti/api-sdk
pnpm install

# vritti-api-nexus
cd ~/Documents/Vritti/vritti-api-nexus
pnpm install

# vritti-web-nexus
cd ~/Documents/Vritti/vritti-web-nexus
pnpm install

# vritti-auth
cd ~/Documents/Vritti/vritti-auth
pnpm install
```

<Tip>
You can install all dependencies at once with a script:

```bash
for dir in quantum-ui api-sdk vritti-api-nexus vritti-web-nexus vritti-auth; do
  echo "Installing $dir..."
  cd ~/Documents/Vritti/$dir && pnpm install
done
```
</Tip>

## Start Database

The backend requires PostgreSQL. Start it with Docker:

```bash
cd ~/Documents/Vritti/vritti-api-nexus

# Start PostgreSQL
docker compose up -d postgres

# Verify it's running
docker compose ps
```

## Run Database Migrations

Push the database schema:

```bash
cd ~/Documents/Vritti/vritti-api-nexus

# Generate and push schema
pnpm db:push
```

## Configure Hosts File

For local development with SSL, add these entries to `/etc/hosts`:

```bash
sudo nano /etc/hosts
```

Add:
```
127.0.0.1 local.vrittiai.com
127.0.0.1 api.local.vrittiai.com
```

## Generate SSL Certificates

Generate local SSL certificates:

```bash
cd ~/Documents/Vritti/vritti-web-nexus

# Install mkcert CA (if not done)
mkcert -install

# Generate certificates
mkcert local.vrittiai.com localhost 127.0.0.1 ::1
```

Move certificates to the expected location or update your project config.

## Verify Setup

Run quick checks:

```bash
# Check Node version
node --version  # v22.x.x

# Check pnpm
pnpm --version  # 9.x.x

# Check Docker
docker compose ps  # postgres should be running

# Check database connection
cd ~/Documents/Vritti/vritti-api-nexus
pnpm db:studio  # Opens Drizzle Studio
```

## Project-Specific Setup

### quantum-ui

```bash
cd ~/Documents/Vritti/quantum-ui

# Start Storybook
pnpm storybook
# Opens at http://localhost:6006

# Or start dev server
pnpm dev
```

### vritti-api-nexus

```bash
cd ~/Documents/Vritti/vritti-api-nexus

# Copy environment file
cp .env.example .env

# Edit with your values
nano .env

# Start development server
pnpm dev
# API available at http://localhost:3002
```

### vritti-web-nexus

```bash
cd ~/Documents/Vritti/vritti-web-nexus

# Copy environment file
cp .env.example .env

# Start with SSL
pnpm dev:ssl
# Opens at https://local.vrittiai.com:3000
```

### vritti-auth

```bash
cd ~/Documents/Vritti/vritti-auth

# Copy environment file
cp .env.example .env

# Start development
pnpm dev
# Served via Module Federation
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="pnpm install fails">
    Clear the cache and retry:
    ```bash
    pnpm store prune
    rm -rf node_modules
    pnpm install
    ```
  </Accordion>

  <Accordion title="Database connection refused">
    Ensure PostgreSQL is running:
    ```bash
    docker compose up -d postgres
    docker compose logs postgres
    ```
  </Accordion>

  <Accordion title="SSL certificate errors">
    Reinstall the local CA:
    ```bash
    mkcert -install
    # Restart your browser
    ```
  </Accordion>

  <Accordion title="Module Federation errors">
    Ensure vritti-auth is running before vritti-web-nexus:
    ```bash
    # Terminal 1
    cd vritti-auth && pnpm dev

    # Terminal 2
    cd vritti-web-nexus && pnpm dev:ssl
    ```
  </Accordion>
</AccordionGroup>
