## Platform Architecture

Vritti is built as a collection of independent projects that work together:

```
┌─────────────────────────────────────────────────────────────┐
│                    VRITTI ECOSYSTEM                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  SHARED PACKAGES (NPM)                                       │
│  ├─ @vritti/quantum-ui    → Component library               │
│  └─ @vritti/api-sdk       → Backend SDK                     │
│                                                              │
│  FRONTEND APPS                                               │
│  ├─ vritti-web-nexus      → Main SaaS dashboard (MF Host)   │
│  ├─ vritti-auth           → Auth micro-frontend (MF Remote) │
│  └─ vritti-landing        → Marketing site                  │
│                                                              │
│  BACKEND                                                     │
│  └─ vritti-api-nexus      → API Gateway (NestJS)            │
│                                                              │
│  INDEPENDENT                                                 │
│  └─ slicingpie            → Equity management app           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## Key Features

<AccordionGroup>
  <Accordion title="Multi-Tenant Architecture">
    - Database-per-tenant isolation
    - Automatic tenant context injection
    - Subdomain-based routing
  </Accordion>

  <Accordion title="Micro-Frontend Architecture">
    - Module Federation for independent deployments
    - vritti-auth loaded dynamically into vritti-web-nexus
    - Shared component library (quantum-ui)
  </Accordion>

  <Accordion title="Modern Authentication">
    - JWT with refresh token rotation
    - Passwordless (WebAuthn/Passkeys)
    - Multi-factor authentication (TOTP, SMS)
    - OAuth (Google, Microsoft)
  </Accordion>

  <Accordion title="Developer Experience">
    - TypeScript everywhere
    - CLAUDE.md files for AI-assisted development
    - Comprehensive API documentation
    - Storybook for component development
  </Accordion>
</AccordionGroup>
