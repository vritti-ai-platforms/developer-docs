# Monorepo Structure

Vritti uses a multi-repo approach with a workspace configuration that links all projects together. This enables consistent tooling while allowing independent deployment of each service.

## Project Categories

### Shared Libraries

These packages are published to npm and consumed by other projects:

<Tabs>
  <Tab title="@vritti/quantum-ui">
    **Purpose:** Shared React component library based on shadcn/ui

    ```bash
    # Location
    /quantum-ui

    # Package name
    @vritti/quantum-ui@0.2.x

    # Key exports
    - UI Components (Button, Card, Dialog, etc.)
    - Form components (with react-hook-form integration)
    - Axios instance with CSRF and auth interceptors
    - Toast notifications
    ```

    **Consumers:**
    - vritti-web-nexus
    - vritti-auth
    - slicingpie
  </Tab>
  <Tab title="@vritti/api-sdk">
    **Purpose:** Shared NestJS modules for backend services

    ```bash
    # Location
    /api-sdk

    # Package name
    @vritti/api-sdk@0.1.x

    # Key exports
    - AuthConfigModule (JWT guards, decorators)
    - DatabaseModule (Drizzle ORM, multi-tenant)
    - LoggerModule (Winston-based logging)
    - Base repositories for common patterns
    ```

    **Consumers:**
    - vritti-api-nexus
    - Future microservices
  </Tab>
</Tabs>

  ```

  ├── src/
  │   ├── db/
  │   │   ├── schema/         # Drizzle ORM schemas
  │   │   └── migrations/     # Database migrations
  │   ├── auth/               # Authentication module
  │   ├── tenants/            # Tenant management
  │   ├── users/              # User management
  │   ├── onboarding/         # Registration flow
  │   └── mfa/                # Multi-factor auth
  └── drizzle.config.ts       # Migration config
  ```

  **Ports:** 3000 (API), 3000/api/docs (Swagger)
</Card>

## VS Code Workspace

The `vritti.code-workspace` file configures VS Code for multi-project development:

```json
{
  "folders": [
    { "path": "quantum-ui", "name": "quantum-ui" },
    { "path": "api-sdk", "name": "api-sdk" },
    { "path": "vritti-api-nexus", "name": "api-nexus" },
    { "path": "vritti-web-nexus", "name": "web-nexus" },
    { "path": "vritti-auth", "name": "auth" },
    { "path": "vritti-landing", "name": "landing" },
    { "path": "slicingpie", "name": "slicingpie" },
    { "path": "developer-docs", "name": "docs" }
  ],
  "settings": {
    "typescript.tsdk": "node_modules/typescript/lib"
  }
}
```

<Tip>
  Open the workspace with: `code vritti.code-workspace`
</Tip>

## Package Management

All projects use **pnpm** for dependency management:

```bash
# Install dependencies in a project
cd vritti-api-nexus
pnpm install

# Link local packages during development
# In quantum-ui:
pnpm build && pnpm link --global

# In consuming project:
pnpm link --global @vritti/quantum-ui
```

## Environment Files

Each project has its own environment configuration:

```
project/
├── .env.example         # Template with all variables
├── .env                  # Local development (git-ignored)
├── .env.staging         # Staging environment
└── .env.production      # Production environment
```

<Warning>
  Never commit `.env` files. Use `.env.example` as the template and document all variables in the [Environment Variables](/getting-started/environment-variables) guide.
</Warning>
